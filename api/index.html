    Руководство по API¶

    Этот раздел — ваш главный справочник по всем компонентам, классам и типам данных AsyncMaxBot SDK, основанный на актуальном коде.

    1. Архитектура и поток данных¶

    graph TD
    A[Пользователь отправляет сообщение] --> B[Max API];
    B --> C{bot.polling};
    C --"JSON-обновление"--> D[Dispatcher];
    D --"Передает в MiddlewareManager"--> E[MiddlewareManager];
    E --"Исполняет Middleware"--> F[Парсинг в Pydantic-модели];
    F --"Создает Context"--> G[Context];
    G --"Проверяет фильтры"--> H{Фильтры: command, text, ...};
    H --"ДА"--> I[Вызов вашего обработчика];
    I --"Использует Context для ответа"--> J{ctx.reply()};
    J --"Вызывает метод Bot"--> K[Bot.send_message];
    K --"HTTP-запрос"--> B;
    B --> L[Пользователь получает ответ];
    H --"НЕТ"--> M[Поиск следующего обработчика];
    1. С помощью bot.polling(dispatcher=dp) бот постоянно запрашивает у Max API новые обновления. 2. Dispatcher получает обновление и передает его в MiddlewareManager. 3. Middleware обрабатывают входящий запрос по цепочке. 4. Данные парсятся в строго типизированные Pydantic-модели (см. раздел "Типы данных"). 5. На основе этих моделей создается удобный объект Context. 6. Dispatcher ищет подходящий обработчик, проверяя сообщение на соответствие его фильтрам. 7. Если фильтры пройдены, вызывается ваша асинхронная функция-обработчик с Context в качестве аргумента.

    2. Класс Bot¶

    Основной класс для взаимодействия с Max API. Рекомендуется использовать как асинхронный контекстный менеджер.

    Инициализация и запуск: 

    async with Bot(token=TOKEN) as bot:
    dp = Dispatcher(bot)
    # ... регистрация обработчиков ...
    await bot.polling(dispatcher=dp)

    Основные методы¶

    • polling(dispatcher, ...): Основной метод для запуска бота.
    • get_me() -> dict: Возвращает информацию о боте в виде словаря.
    • send_message(chat_id, text, ...): Отправляет текстовое сообщение.
    • edit_message(message_id, text, ...): Редактирует сообщение.
    • delete_message(message_id): Удаляет сообщение.
    • send_photo(chat_id, photo, ...): Отправляет фото. В photo можно передать путь к файлу, io.BytesIO или ID файла.
    • send_document(chat_id, document, filename=None, ...): Отправляет документ.
    • get_chat_members(chat_id): Возвращает список участников чата.
    • send_action(chat_id, action): Отправляет действие в чат (например, typing).

    3. Класс Dispatcher¶

    Диспетчер обрабатывает обновления, управляет middleware и вызывает нужные обработчики.

    from maxbot import Dispatcher

dp = Dispatcher(bot)
    - @dp.message_handler(*filters): Декоратор для регистрации обработчика сообщений. - dp.middleware_manager: Экземпляр MiddlewareManager, к которому можно добавлять middleware.

    4. Middleware¶

    Middleware позволяют выполнять сквозную логику для всех обновлений.

    Пример использования: 

    from maxbot.middleware import MiddlewareManager, LoggingMiddleware, ThrottlingMiddleware

manager = MiddlewareManager()
manager.add_middleware(LoggingMiddleware())
manager.add_middleware(ThrottlingMiddleware(rate_limit=1.0)) # 1 сообщ./сек
dp.middleware_manager = manager

    Встроенные Middleware¶

    • LoggingMiddleware: Логирует информацию о входящих сообщениях.
    • ErrorHandlingMiddleware: Перехватывает ошибки в обработчиках, чтобы бот не падал.
    • ThrottlingMiddleware: Ограничивает частоту сообщений от одного пользователя.
    • AntispamMiddleware: Блокирует одинаковые сообщения от пользователя за короткий промежуток.
    • MetricsMiddleware: Собирает метрики (время работы, кол-во сообщений/ошибок).
    • UserTrackingMiddleware: Отслеживает активных пользователей.

    5. Класс Context¶

    Ключевой объект, с которым вы работаете в обработчиках. Содержит все данные об обновлении и удобные методы для ответа.

    Импорт: 

    from maxbot import Context

    Основные атрибуты¶

    • user: Объект User. Имя пользователя доступно как ctx.user.name.
    • chat: Объект Chat.
    • message: Объект Message.
    • text: Текст сообщения (str | None).
    • attachments: Список вложений (list[BaseAttachment] | None).
    • has_attachments: True, если есть хоть одно вложение.
    • user_id, chat_id: ID пользователя и чата.

    Основные методы¶

    • reply(text, ...): Ответить в тот же чат.
    • answer(text, ...): Псевдоним для reply.

    6. Фильтры¶

    Фильтры решают, будет ли вызван обработчик. Их можно комбинировать.

    Импорт и использование: 

    from maxbot.filters import command, text, regex, attachment_type

@dp.message_handler(command("start"))
async def start_handler(ctx: Context): ...

@dp.message_handler(text("привет", exact=False))
async def hello_handler(ctx: Context): ...

    Основные фильтры¶

    • command(names): Срабатывает на команды.
    • text(texts, exact=True, ignore_case=False): Срабатывает на текст.
    • regex(pattern): Проверяет текст по регулярному выражению.
    • attachment_type(types): Проверяет наличие вложения определенного типа (image, video, file и т.д.).
    • has_attachment(has=True): Проверяет наличие (True) или отсутствие (False) любого вложения.
    • user_filter(user_ids): Срабатывает, если сообщение от указанного пользователя.
    • time_filter(start_hour, end_hour): Срабатывает в определенный промежуток времени.
    • custom_filter(func): Ваш собственный фильтр на основе функции lambda ctx: ....

    7. Типы данных¶

    SDK использует Pydantic для валидации. Основные модели импортируются из maxbot.

    • User: Пользователь.
    • user_id: int
    • name: str
    • first_name: str | None
    • last_name: str | None
    • Chat: Чат.
    • chat_id: int
    • chat_type: str - (private или group)
    • Message: Сообщение. Содержит sender: User, recipient: Chat, body: MessageBody.
    • MessageBody: Тело сообщения. Содержит text и attachments.
    • BaseAttachment: Базовая модель для вложений. Содержит type, url, filename, size и т.д.
